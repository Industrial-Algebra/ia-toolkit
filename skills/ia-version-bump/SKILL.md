# IA Version Bump

## Overview

Bump versions consistently across a multi-crate IA workspace. Handles the
three places versions appear: workspace package declaration, internal dependency
entries, and the CHANGELOG. Always verifies with `cargo check` afterward.

## When to Use

- Preparing a new release (e.g., 0.5.0 → 0.6.0)
- Post-release dev bump (e.g., 0.5.0 → 0.6.0-dev after publishing)
- User says "bump version" or "prepare release"

## Prerequisites

Read these files before starting:
- Workspace `Cargo.toml` — `[workspace.package]`, `[workspace.dependencies]`
- All sub-crate `Cargo.toml` files — check for inline versions (not `.workspace = true`)
- `CHANGELOG.md` — current version header, format style
- `Cargo.lock` — will be regenerated by `cargo update` or `cargo check`

## Workflow

### 1. Determine Version Strategy

```
New version from user input, or derive:
  PATCH (0.5.0 → 0.5.1) — bug fixes, no API changes
  MINOR (0.5.0 → 0.6.0) — new features, backward compatible
  MAJOR (0.5.0 → 1.0.0) — breaking API changes
  PRE   (0.5.0 → 0.6.0-dev) — pre-release suffix
```

### 2. Locate All Version References

Search the workspace for every occurrence of the current version:

```bash
cd $WORKSPACE_ROOT
CURRENT=$(grep '^version = "' Cargo.toml | head -1 | sed 's/.*"\(.*\)".*/\1/')
echo "Current: $CURRENT"

# Find all version references (case-insensitive, whole word)
rg "\"$CURRENT\"" --type toml
```

This reveals:
- `[workspace.package] version = "0.5.0"` — the source of truth
- `karpal-core = { version = "0.5.0", path = "karpal-core" }` — internal deps
- Any stray inline `version = "0.5.0"` in sub-crates

### 3. Update Workspace Package Version

**Single edit** — the workspace-level version in `[workspace.package]`:

```toml
[workspace.package]
version = "0.5.0"   # ← change to "0.6.0"
```

After this change, all sub-crates with `version.workspace = true` automatically
pick up the new version. No per-crate edits needed for those.

### 4. Update Internal Dependency Versions

In `[workspace.dependencies]`, every entry with a path also has a version:

```toml
# Before:
karpal-core = { version = "0.5.0", path = "karpal-core" }
karpal-proof = { version = "0.5.0", path = "karpal-proof" }

# After (all bumped together):
karpal-core = { version = "0.6.0", path = "karpal-core" }
karpal-proof = { version = "0.6.0", path = "karpal-proof" }
```

**Batch replace:** Use `rg` to find all occurrences, then replace in one pass.

**Double-check:** External dependencies (like `tokio = { version = "1.0" }`) do NOT get bumped unless you're intentionally upgrading them. Only bump IA internal deps.

### 5. Update CHANGELOG.md

Insert a new version header at the top, following the existing format:

```markdown
## [0.6.0] — Unreleased

### Added
-

### Changed
-

### Fixed
-
```

Preserve the existing entry (it becomes the previous version). Use the same
section structure the file already uses.

### 6. Check for Stray Inline Versions

Some sub-crates may declare `version = "0.5.0"` inline (not `.workspace = true`).
These MUST be found and updated:

```bash
# Find sub-crate Cargo.toml files with inline version
find . -name Cargo.toml -not -path './Cargo.toml' \
  -exec grep -l '^version = "' {} \;
```

Update each to match the new workspace version. If a sub-crate intentionally
diverges from the workspace version, document the reason in a comment.

### 7. Verify

```bash
cd $WORKSPACE_ROOT
cargo check --all-features
cargo test --all-features  # if tests exist
```

**Fix any compilation errors before declaring success.** Version mismatches
between `[workspace.dependencies]` and `version.workspace = true` in sub-crates
will cause hard-to-diagnose resolution failures. Cargo may silently select an
older published version instead of the local path — always verify.

### 8. Commit

```bash
git add -A
git commit -m "chore: bump version to 0.6.0

- Workspace package version updated
- Internal dependency versions synchronized
- CHANGELOG entry added"
```

## Semver Decision Guide

| Change Type | Bump | Example |
|---|---|---|
| New public function/item | MINOR | 0.5.0 → 0.6.0 |
| New feature gate | MINOR | 0.5.0 → 0.6.0 |
| Bug fix, no API change | PATCH | 0.5.0 → 0.5.1 |
| Doc-only change | PATCH | 0.5.0 → 0.5.1 |
| New public type | MINOR | 0.5.0 → 0.6.0 |
| Removed public item | MAJOR | 0.5.0 → 1.0.0 |
| Changed type signature | MAJOR | 0.5.0 → 1.0.0 |
| Changed feature gate behavior | MAJOR | 0.5.0 → 1.0.0 |
| Pre-release suffix | PRE | 0.5.0 → 0.6.0-dev |

## Common Pitfalls

| Mistake | Fix |
|---|---|
| Bumped workspace version but not internal dep versions | Search for all occurrences of the old version string |
| Forgot to bump a sub-crate with inline `version = "X.Y.Z"` | Use `find` + `grep` to locate inline versions |
| Bumped external dep versions accidentally | Only change IA internal deps (those with `path = "..."`) |
| Didn't update CHANGELOG | Add new header before existing content |
| Cargo resolves to published version instead of local path | Check that `version` in `[workspace.dependencies]` matches `workspace.package.version` |
| Committed before `cargo check` passes | Always verify last |
